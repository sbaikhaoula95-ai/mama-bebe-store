# Hnina Google Sheets Webhook

The backend POSTs each order as a single flat row to a Google Apps Script web app, which appends it to the `orders` sheet.

## One-time setup

1. **Create a Google Spreadsheet** (e.g. `OrdersHNINAStore`) at sheets.google.com.
2. **Extensions → Apps Script** → delete the default code → paste the whole contents of `Code.gs` from this folder → save.
   If using a standalone Apps Script project, paste the spreadsheet ID into
   `SPREADSHEET_ID` in `Code.gs`.
3. **Deploy → New deployment → type "Web app"**:
   - Description: `hnina-orders-webhook`
   - Execute as: **Me**
   - Who has access: **Anyone**
   - Click **Deploy** and copy the `/exec` URL.
4. **Backend env**:
   ```
   SHEET_WEBHOOK_URL=https://script.google.com/macros/s/XXXXXXXXXXXX/exec
   ```
   No secret is required.
5. **Smoke test**: in the Apps Script editor select `testWebhook` → ▶ Run. A test row should appear in the `orders` tab.

## Sheet columns

The script auto-creates a single `orders` tab with this header row (do not change the order):

| date | orderid | country | name | phone | product | sku | quantity | total price | currency | status |
|------|---------|---------|------|-------|---------|-----|----------|-------------|----------|--------|

- `date` — DD/MM/YYYY in Morocco time.
- `orderid` — `HNINA-YYYYMMDD-XXXX` generated by the backend.
- `country` — always `MOROCCO`.
- `phone` — local Moroccan format `06XXXXXXXX` / `07XXXXXXXX`. Stored as text so leading zeros survive.
- `product` / `sku` / `quantity` — slash-separated, one slot per ordered item, in the same order across the three columns. Example: `حنينة ماما/حنينة جذور`, `HNM-7281/HNJ-3469`, `2/1`.
- `total price` — number in MAD.
- `status` — left empty for the operations team to fill (e.g. `called`, `confirmed`, `shipped`, `delivered`).

## Re-deploying after changes

Apps Script ties each `/exec` URL to a specific deployment. If you edit `Code.gs`, click **Deploy → Manage deployments → ✎ (edit)** on the existing one and pick **New version**. The same URL keeps working — no need to update the backend env.

After fixing `SHEET_WEBHOOK_URL`, retry failed/pending orders from the backend:

```sh
curl -X POST -H "X-Webhook-Secret: $WEBHOOK_SHARED_SECRET" \
  "https://api.hnina.shop/api/orders/sheets/retry-failed?limit=50"
```
