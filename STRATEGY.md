## 0. Core strategy (why this wins)

1. **In COD Morocco, profit is decided AFTER the click** — by **confirmation rate** (answering the phone) and **delivery rate** (taking the box). Returns of 30–50% kill stores; 10–15% prints money.
2. **We sell to desperate, high-insecurity buyers** (stretch marks, sleepless baby, irritated baby skin) → they answer the phone, receive the box, reorder. Maxes confirmation + delivery + LTV.
3. **We brand products that already exist & are trusted in Morocco** (cheap in herboristerie/cooperatives), **topical, safe, non-ingestible, easy to package**. Buy cheap, sell premium, ship local 24–48h.
4. **We win on TRUST**, not price: science + ingredients + certificates + doctor/midwife authority + heavy social proof + heritage (beldi) + a fearless guarantee.

### 🎯 North-star metrics (the only numbers that matter)

| Metric | Morocco COD average | **Hnina target** |
|---|---|---|
| Confirmation rate | 55–65% | **75–82%** |
| Delivery rate | 45–55% | **70–80%** |
| RTO (return-to-origin) | 30–50% | **< 25%** |
| Repeat (90d) | 5–10% | **20–30%** |

> The fast Darija confirmation call (within minutes of order) is the **#1 lever** for all four. The lineup below was chosen *because* it maxes those numbers.

**Single price ladder (by total quantity in cart, mix-and-match any products):**

### 9.4 Checkout flow (popup → timed upsell → thank-you)
**Step 1 — Checkout popup form (only "checkout"):**
- **Order summary** (items, qty, total MAD).
- **5 fields:** `الاسم الكامل` + `رقم الهاتف` + `أعيدي رقم الهاتف` + `المدينة` + `العنوان الكامل`.
- Phone helper: *مثال: 0612345678* (starts with 0).
- **Validation:** Moroccan mobile `^0[67]\d{8}$` (10 digits, 06/07), phone confirmation must match, city/address required. Inline Darija errors if invalid.
- CRO in popup: COD reassurance ("ما تخلصي والو دابا"), 24–48h, guarantee badge, "+10,000 moms", no-spam note, high-contrast CTA **"تأكيد الطلب"**.
- Honeypot + submit throttle (anti-fake-order).

**Step 2 — Single timed UPSELL (the ONLY discount, fires AFTER form validation + CTA):**
- Show **exactly ONE product**, **not in the cart**, at a **99 MAD** discount.
- **Countdown 10–15s**, two buttons: **"نعم، زيدها (99 درهم)"** / **"لا، شكراً"**.
- Accept → add as a discounted add-on line → proceed. Decline OR timeout → proceed with original items only.
- **Recommendation matrix (rule-based, new lineup):**

  | Cart contains | Upsell ONE (at 99) |
  |---|---|
  | Hnina Mama only | **Hnina Lila** (next motherhood stage) |
  | Hnina Lila only | **Hnina Mama** (high-margin lead — "take care of mom too") |
  | Hnina Calm only | **Hnina Lila** (same baby, broader ritual) |
  | Mama + Lila | **Hnina Calm** (complete the trio) |
  | Mama + Calm | **Hnina Lila** |
  | Lila + Calm | **Hnina Mama** |
  | All 3 already | extra **Hnina Mama** at 99 (refill) — or skip |
  | (fallback) | **Hnina Mama** |

**Step 3 — Submit → `/api/order` → webhook to Google Sheet → redirect `/merci?order=<id>`.** Fire Pixel/TikTok `Lead` (generic event name — ad-safety).

**Webhook → Sheet columns:** orderId, timestamp, name, phone, items (name×qty×price), upsellAccepted (y/n), upsellItem, subtotal, total, leadProduct/sourcePage, utm_*, pageURL, userAgent. (You call from the Sheet; capture address on the call.)

### 9.5 Thank-you page (`/merci`) — CRO + lift confirmation/delivery
- Success check + **"طلبك توصلنا!"** + order ID + **final order summary** (includes the upsell item **only if accepted**; same data sent to Sheet).
- **"شنو غادي يوقع دابا" — 3 steps:** (1) نعيطو ليك باش نأكدو ونأخذو العنوان · (2) كنصيفطو 24–48 ساعة · (3) كتخلص عند الاستلام.
